# 邊界條件(Edge Cases)

> **目的**:在實作前把「不是 happy path 的情境」列清楚,逼自己想清楚怎麼處理。
> **負責人**:負責該模組的工程師
> **Review**:另一位工程師 + QA
> **核心**:列出來的每個情境,都要明確寫怎麼處理,不能只是「待確認」。

---

## 1. 輸入邊界

| 情境 | 預期處理 | 測試案例 |
|------|---------|---------|
| 空字串 | 視欄位:必填回 400、選填當未提供 | "" |
| Null / undefined | 必填回 400 | null |
| 超長字串 | 截斷 or 拒絕(視欄位) | 10001 字元 vs 上限 10000 |
| 特殊字元 | 不轉義會 XSS / SQL injection | <script>、'; DROP-- |
| Unicode / Emoji | 正常支援 | 你好、😀、ﷺ |
| 全形/半形 | 正規化(NFKC) | 1234 vs 1234 |
| 前後空白 | 自動 trim | "  abc  " |
| 不同編碼 | 統一 UTF-8 | BOM、Big5 上傳 |

## 2. 數值邊界

| 情境 | 處理 |
|------|------|
| 0 | 視情況允許 / 拒絕 |
| 負數 | 通常拒絕(金額、數量) |
| 極大值 | 上限檢查(避免 overflow、UI 爆版) |
| 浮點精度 | 金額用 decimal,不用 float |
| 除以 0 | 顯式檢查 |

## 3. 集合邊界

| 情境 | 處理 |
|------|------|
| 空陣列 | 視欄位(購物車項目不能空) |
| 單一元素 | 走一般流程 |
| 超大量(1000+) | 分頁 / 拒絕 / 批次 |
| 重複元素 | 視情況去重 |
| 順序依賴 | 是否允許亂序 |

## 4. 時間邊界

| 情境 | 處理 |
|------|------|
| 跨時區 | 儲存 UTC、顯示依使用者時區 |
| 夏令時間 | 用標準時區函式庫 |
| 閏年 / 2/29 | 測試案例覆蓋 |
| 過去時間 | 視情境拒絕(預約) |
| 未來時間 | 視情境允許(排程) |
| 時間倒退 | NTP 異常時,系統時間可能跳動 |

## 5. 並發與競爭

| 情境 | 處理 |
|------|------|
| 同一資源同時更新 | 樂觀鎖(version)或悲觀鎖 |
| 重複請求 | Idempotency key |
| 雙擊送出 | 前端 disable + 後端 idempotent |
| Race condition(讀後寫) | 交易或 CAS |
| 訊息重複消費 | 訊息處理 idempotent |
| 訊息亂序 | 用 sequence number 或不依賴順序 |

## 6. 容量與效能邊界

| 情境 | 處理 |
|------|------|
| 大檔案上傳 | 分塊、限制大小、串流而非全載入記憶體 |
| 大 result set | 分頁、游標 |
| N+1 查詢 | 預先 join 或 batch load |
| 長交易 | 拆短、避免鎖太久 |
| 慢查詢 | 加索引、加超時 |

## 7. 失敗與不可用

| 情境 | 處理 |
|------|------|
| DB 不可用 | 回 503、不要 retry storm |
| 第三方 timeout | 重試 + 斷路器 |
| 部分失敗(分散式) | Saga / 補償交易 |
| 寫成功但回應失敗 | Idempotency key 讓 client 安全重試 |
| 訊息送出後 crash | Outbox pattern |

## 8. 認證與授權邊界

| 情境 | 處理 |
|------|------|
| Token 過期(剛剛) | 401 + refresh flow |
| Token 即將過期 | 主動 refresh |
| 多裝置登入 | 是否互斥?policy 寫清楚 |
| 權限變更後現有 session | 即時失效 vs 等過期 |
| 越權存取(IDOR) | 每次操作驗證 ownership |

## 9. 業務規則邊界

> 列出該業務的特殊情境。

| 情境 | 處理 |
|------|------|
| 訂單剛好在付款時被取消 | 鎖定狀態、不允許並行 |
| 庫存剛好為 0 | 友善訊息 + 提供替代品 |
| 優惠券到期前 1 秒使用 | 用伺服器時間判斷 |
| 退款金額大於原訂單 | 拒絕 |
| 跨年訂單編號 | 不要用日期當 PK |

## 10. UI / UX 邊界

| 情境 | 處理 |
|------|------|
| 載入中 | Skeleton / spinner |
| 空狀態 | 引導使用者下一步 |
| 錯誤狀態 | 明確訊息 + 重試按鈕 |
| 離線 | 偵測 + 提示 |
| 字串過長 | 截斷 + tooltip |
| 多語系字串長度差異 | 彈性版面,不要寫死寬度 |
| 無障礙 | 鍵盤操作、screen reader |

## 11. 安全邊界

| 情境 | 處理 |
|------|------|
| SQL injection | 參數化查詢 |
| XSS | 輸出 escape、CSP |
| CSRF | Token / SameSite cookie |
| Open redirect | 白名單 |
| 暴力破解 | Rate limit + 帳號鎖定 |
| 敏感操作 | 二次確認 / 二次驗證 |
